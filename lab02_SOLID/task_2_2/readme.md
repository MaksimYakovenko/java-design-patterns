# Аркадна гра

## Завдання

### Завдання 1: Виявлення порушень принципів SOLID

1) Порушено принципи DRY (дублювання коду) (незначий, але є дефект)

- У `addDataMS()`, текст такий самий як в `updateDataMS()`
```bash
System.out.println("update data in MSDataBase");
```
2) Порушено принип SRP (одна відповідальність)
- Клас `MSDataBase` виконує декілька різних обов'язків:
    - Містить бізнес-логіку (умови ділення на 3, цикл)
    - Керує викликами методів БД (`addDataMS`, `updateDataMS`, `getDataMS`)

3) Dependency Inversion порушено
- `MSDataBaseClient` напряму залежить від конкретного класу `MSDataBase`, а мав би залежати від абстракції (інтерфейсу або абстрактного класу).
