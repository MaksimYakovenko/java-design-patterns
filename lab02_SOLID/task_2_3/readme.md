## Завдання

### Завдання 3: Транспортні засоби

1) LSP порушено, бо:
   - LSP каже: якщо `Bicycle` є `Vehicle`, то його можна підставити всюди, де очікують `Vehicle`, і програма не повинна ламатися.
   - Але в базовому класі `Vehicle` є метод `drive()`. Для машини це ок, а для велосипеда не дуже.
2) ISP порушено, бо:
   - Через те, що в `Vehicle` є `drive()`, велосипед вимушено отримує метод, який йому не підходить (йому потрібен `ride()`), і мусить або: тримати “зайвий” метод, або ламати поведінку через exception.

Найправильніше було б:
1) Зробити базовий клас/інтерфейс без “руху”, а рух винести в окремі інтерфейси:
```bash
interface Drivable {
    void drive();
}

interface Ridable {
    void ride();
}

```
Тоді:
```bash
Car implements Drivable
```

```bash
Bicycle implements Ridable
```
Спільні поля (name, brand, maxSpeed) можна залишити в abstract class Vehicle.
Це прибирає і LSP, і ISP проблему, бо велосипед більше не “зобов’язаний” мати drive().
